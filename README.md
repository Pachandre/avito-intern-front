# Система управления объявлениями для модерации

Веб-приложение для модерации объявлений на платформе Авито. Разработано в рамках тестового задания для стажёра Frontend (осенняя волна 2025).

## Технологический стек

### Обязательные технологии

-   **Node.js** v20
-   **React** v18+
-   **react-router-dom** - маршрутизация приложения
-   **TypeScript** - типизация кода

### Дополнительные библиотеки

-   **Ant Design** - UI-библиотека компонентов
-   **React Query** - управление серверным состоянием и кэширование
-   **Recharts** - визуализация данных (графики и диаграммы)
-   **use-local-storage** - синхронизация состояния с localStorage
-   **react-keybind** - обработка горячих клавиш
-   **Vite** - сборщик и dev-сервер

## Требования

-   Node.js v20 или выше
-   npm или pnpm
-   Docker и Docker Compose (опционально, для контейнеризированного запуска)

## Запуск

### Локальный запуск без Docker

#### 1. Запуск Backend API

```bash
cd backend

npm install

npm start
```

API будет доступен на `http://localhost:3001`

#### 2. Запуск Frontend

```bash
cd frontend

npm install

npm run dev
```

Frontend будет доступен на `http://localhost:5173`

## Запуск через Docker

### Быстрый запуск всего приложения

```bash
docker-compose up
```

Сервисы будут доступны:

-   Frontend: `http://localhost:5173`
-   Backend API: `http://localhost:3001`

### Остановка контейнеров

```bash
docker-compose down
```

### Пересборка контейнеров

```bash
docker-compose up --build
```

## Обоснование выбора библиотек

### Ant Design

**Выбор:** UI-библиотека компонентов для React

**Обоснование:** Много компонентов, поддержка темной / светлой темы, реактивность, типизация

### React Query (TanStack Query)

**Выбор:** Библиотека для управления асинхронными данными и кэширования

**Обоснование:** Реактивность, типизация, поддержка AbortSignal

### Recharts

**Выбор:** Библиотека для построения графиков и диаграмм

**Обоснование:** Реактивность, много компонентов, гибкая настройка

### use-local-storage

**Выбор:** React hook для работы с localStorage

**Обоснование:** Сохранение количества объявлений на странице в localStorage

### react-keybind

**Выбор:** Библиотека для обработки горячих клавиш

**Обоснование:** Легковесность, типизация

### Реализованные дополнительные требования

-   Горячие клавиши (A, D, ←, →, /)
-   Тёмная тема с сохранением в localStorage
-   URL-синхронизация фильтров
-   Контейнеризация (Docker)
-   Прерывание запросов при переходе между страницами
-   TypeScript для типобезопасности
-   Prettier для форматирования кода
